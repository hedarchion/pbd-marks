## Project Overview

This is a zero-dependency, single-file static webapp for entering PBD (school assessment) marks. It is deployed via GitHub Pages and designed mobile-first.

## Architecture

- **Single-file app**: All markup, styles, and logic live in `index.html`. There is no build step, no bundler, and no external dependencies.
- **Data model**: The canonical student roster is hardcoded in the `STUDENTS` object (lines ~266–272). Each class key maps to an ordered array of student names.
- **Persistence**: Marks are stored in `localStorage` under the key `pbd_marks_data`. The currently selected class is stored under `pbd_current_class`.
- **Grade scale**: Hardcoded in `GRADE_SCALE` (lines ~274–281) and documented in `grades.md`.
- **Export**: The "Export Markdown" button generates a markdown table that matches the format of `CLASS_MARKS.md`.

## Development

- **Preview locally**: `python3 -m http.server 8000` (or simply open `index.html` in a browser).
- **No tests / linting**: There is no test suite or linter configured.
- **Deployment**: Push to `main`; GitHub Pages serves `index.html` from the repo root.

## Data Workflow

1. **Student list updates**: Edit the `STUDENTS` object directly in `index.html`. The `CLASS_MARKS.md` file is generated by the app via Export and should not be hand-edited for roster changes.
2. **Source data**: PDFs (`1D.pdf`, `1E.pdf`, etc.) and `PBD.xlsx` are reference files containing original class lists. They are gitignored for cleanliness but remain tracked in the repo.
3. **Grading reference**: `grades.md` contains the official percentage-to-grade mapping. If the scale changes, update both `grades.md` and the `GRADE_SCALE` array in `index.html`.
